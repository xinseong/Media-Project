# DA3 Giant GPU Memory Usage Analysis

본 문서는 DA3 Giant (ViT-Giant) 모델을 사용하여 고해상도(1920x1440) AR 세션 데이터를 처리할 때 발생하는 GPU 메모리(VRAM) 부하를 분석하고, 최적화 방안을 제시합니다.

## 1. 모델 사양 및 데이터 특성
- **모델**: DA3 Giant (Vision Transformer 기반)
- **입력 해상도**: 1920 x 1440 (약 2.76M Pixels)
- **하드웨어 정보**: Tesla V100 32GB (기존 32GB VRAM 제한)

## 2. 메모리 계산 수식 (Inference 기준)

GPU 메모리 총량($M_{total}$)은 다음과 같이 정의됩니다:

$$M_{total} \approx M_{weights} + (Batch\_Size \times (M_{input} + M_{activations}))$$

### 상세 항목 설명
1. **$M_{weights}$ (정적 부하)**:
   - 모델 가중치 데이터. DA3 Giant (FP16/BF16) 기준 **약 2.6 GB**의 고정 메모리를 점유합니다.
2. **$M_{input}$ (입력 텐서)**:
   - $1920 \times 1440 \times 3 \times 4$ bytes (Float32 기준) $\approx$ **약 33 MB**.
3. **$M_{activations}$ (동적 부하/Peak)**:
   - Transformer 레이어를 통과하며 생성되는 중간 연산 결과물.
   - **ViT 기하급수적 부하**: 이미지를 패치(14x14)로 나눌 때 발생하는 토큰 수($N$)에 비례하여 증가합니다.
   - **경험적 상수**: 화소(Pixel) 하나당 약 **2.7~3 KB**의 메모리가 필요합니다.

## 3. 해상도별 메모리 요구량 비교 (단일 이미지)

| 항목 | 원본 (1920x1440) | 1/2 다운샘플링 (960x720) | 비고 |
| :--- | :--- | :--- | :--- |
| **패치 토큰 수($N$)** | 13,974개 | 3,468개 | **4배 차이** |
| **가중치 메모리** | 2.6 GB | 2.6 GB | 고정 |
| **활성화 맵(Peak)** | **약 7.4 GB** | **약 1.85 GB** | 해상도 비례 ($N \times D \times \alpha$) |
| **총 요구 VRAM** | **약 10.0 GB** | **약 4.5 GB** | 이미지 1장 처리 시 |

---

## 4. 처리 방식별 시나리오 (이미지 10장 기준)

서버의 가용 VRAM **32GB**를 기준으로 분석한 결과입니다.

### Case A: 배치 처리 (Batch Processing)
- **수식**: $2.6GB + (10 \text{장} \times 7.4GB) = \mathbf{76.6 GB}$
- **결과**: **Failure (OOM)**
- **분석**: 라이브러리가 10장을 한꺼번에 VRAM에 올리거나 연산을 시도할 경우 하드웨어 한계를 크게 초과합니다.

### Case B: 다운샘플링 배치 처리 (Downsampled Batch)
- **수식**: $2.6GB + (10 \text{장} \times 1.85GB) = \mathbf{21.1 GB}$
- **결과**: **Success (안전)**
- **분석**: 해상도를 줄여 한 번에 처리 가능하나, 컵의 미세한 기하학적 정보가 손실될 우려가 있습니다.

### Case C: 순차적 처리 (Sequential Processing) - **추천**
- **수식**: $2.6GB + (1 \text{장} \times 7.4GB) \times 10 \text{회 반복} = \mathbf{10.0 GB \text{ 유지}}$
- **결과**: **Success (매우 안정)**
- **분석**: 한 장씩 처리하고 `empty_cache()`를 실행하여 메모리를 관리합니다. 원본의 정밀도를 100% 유지하면서 이미지 장수에 관계없이 무한히 확장이 가능합니다.

## 5. 최종 권고안

팀의 정밀도 요구사항을 고려하여 **"순차적 처리(Sequential Processing)"** 방식을 채택합니다.
- **장점**: 원본 해상도 유지로 인한 부피 측정 정확도 극대화.
- **성능**: V100 기준 1장당 ~2초 내외, 10장 처리 시 약 20초 소요 (충분히 수용 가능한 수준).
